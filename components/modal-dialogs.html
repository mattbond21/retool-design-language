A modal dialog is used to direct a user’s attention to a specific action, information, or a set of actions. It typically lies above a page’s content and has a background opacity to focus the user’s attention on the modal.

The content dictates the height of the modal and can expand to accommodate more content. The modal should be anchored 80px from the top and increase in height until it is 80px from the bottom of the screen. If the content extends beyond this max-height, the modal body becomes scrollable.

The width of the dialog is available in four sizes:

  • Small: 480px
  • Medium: 640px
  • Large: 800px
  • X-large: 960px

Types and anatomy

Basic modal

  1. Header: This should include a title that describes what triggered the modal or what the context of the modal is. If one of the actions in the modal footer is a “Cancel” action, you do not need to have the close icon in the header for redundancy.
  2. Body: The body of the modal will be dependent on the context with which it was triggered. This is a space to customize your content within the constraints of the different modal types listed below.
  3. Actions: Contains a maximum of two right aligned buttons. The main action should be a primary button to confirm the state of the dialog. The secondary button is used to cancel the current dialog state and dismiss the modal.

Creation modal

  1. Header: This should include a title that describes what triggered the modal or what the context of the modal is. If one of the actions in the modal footer is a “Cancel” action, you do not need to have the close icon in the header for redundancy.
  2. Input field: The active state of an input field where the input label and user generated information are both visible.
  3. Text area: The empty and inactive state of a text area showing the label.
  4. Actions: Contains a maximum of two right aligned buttons. The main action should be a primary button to confirm the state of the dialog. The secondary button is used to cancel the current dialog state and dismiss the modal.

Selection modal

  1. Header: This should include a title that describes what triggered the modal or what the context of the modal is. If one of the actions in the modal footer is a “Cancel” action, you do not need to have the close icon in the header for redundancy.
  2. Tabs: If appropriate, tabs can be used within a modal to separate content into logical sections. If tabs are used, the height of the modal should not adjust when switching between tabs.
  3. Row item: State when an item is not selected in the table.
  4. Row selection: State when an item is selected in the table.
  5. Actions: Contains a maximum of two right aligned buttons. The main action should be a primary button to confirm the state of the dialog. The secondary button is used to cancel the current dialog state and dismiss the modal.

Benefits modal

Use to introduce existing users to a new experience. The modal is an entry point that focuses all of the user’s attention on a large or impactful update.

  1. Illustration: Should be composed of metaphorical imagery that relates to the content or benefit (see best practices below).
  2. Title: The title should provide a concise overview of the feature benefit.
  3. Message: Supporting information about the feature.
  4. Actions: Contains a maximum of two centered buttons. The main action should be a primary button that uses self-describing action verbs like ‘Get started’. If the secondary button is simply a ‘Dismiss’ action, remove it in favor or focusing on the positive primary button.
  5. Close: Due to the nature of the actions in a benefits modal, we prefer to use the close icon pattern her to dismiss the modal.
