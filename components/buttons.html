Use buttons to trigger actions and links. Buttons can contain a combination of a clear label and an icon while links are always text. Buttons are typically placed throughout the UI — modal dialogs, forms, and toolbars.

Types and anatomy

  1. Primary button (high emphasis): Primary buttons have more emphasis, as they use a color fill to stand out on the page, and are used as the most important action on the page.
  2. Secondary button (medium emphasis): Secondary buttons are used for more emphasis than text buttons but are not the most important action on the page.
  3. Text button (low emphasis): Text buttons are typically used for less important actions.
  4. Toggle button: Toggle buttons group a set of actions together. They’re used when switching between similar view types of content where an icon represents each option, like a list or grid view. If you’re looking to use labels for navigation purposes, please use the tab component instead.
  5. Icon button: Icon buttons are used in toolbars when there are too many actions to display with labels. These are usually the least important actions on the page, and should not have the primary button style applied. If there are more than four actions to be displayed together, use an overflow button and menu to display the remaining actions.

Hierarchy

Only one primary action

A layout should contain a single prominent button that guides users with the most important action on the page. Other buttons have less importance in the hierarchy and appropriately toned down.

Obvious, easy, and possible actions

An app can show more than one button in a layout at a time, so a high-emphasis button can be accompanied by medium- and low-emphasis buttons that perform less important actions. When using multiple buttons, ensure the available state of one button doesn’t look like the disabled state of another.

Labeling

All button labels are sentence case. They should be as short as possible, while clearly explaining to users what happens when they click the button.

Sizes

  1. Small (32px): Used in tables or smaller spaces where the action isn’t competing for the users attention.
  2. Medium (40px): The default button size for most actions on the page.
  3. Large (48px): Should be used sparringly, and only when the medium button isn’t prominent enough.

States

  1. Default: Buttons should look provide the affordance of being clickable in their default state.
  2. Hover: We darken the background, text color, or border color to indicate interactivity.
  3. Disabled: We reduce the opacity of the button when it is disabled. When the button becomes enabled again, the button returns to it's default state.
