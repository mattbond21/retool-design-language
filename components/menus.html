Menus allow users to make a selection from multiple options in a list. Menus are less prominent in the UI and should be used for infrequent actions, when the UI is showing too many actions in a toolbar, or when an overflow is needed due to space constraints.

Behavior

Menus are only triggered when clicking or tapping (not on hover) glyph buttons, selection controls, or input fields.

Menus should appear in front of all other permanent UI elements when active. When ordering menu items, the most important items should be shown at the top of the menu.

Types and anatomy

Text list

 

  1. The most common actions are positioned at the top of the menu
  2. Destructive or least common actions should be separated at the bottom of the list

Text and icon list

 

  1. Icon and text pairing to cerate emphasis on the action importance
  2. Hover state

Text with selection state

 

  1. When using a selection group, there is an optional section heading added for clarity.
  2. Selected state
  3. Hover state

User profile

 

  1. User details are not clickable and don’t have a hover state
  2. Prioritized actions based on user importance
  3. Menu divider
  4. Secondary actions grouped together
  5. Least common or destructive actions should be separated at the bottom of the list
