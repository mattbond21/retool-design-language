Depth is a device that we use in our apps to create hierarchy a sense of layering in the UI. By showing depth we allow surfaces to move in front of and behind other surfaces. This creates spatial relationships between elements for emphasis and focus. Depth can be depicted using shadows or other visual cues, such as surface fills or opacities.

Types and usage

  1. X-Small: Used on button hover states, or very small interactive elements to create separation from the content behind.
  2. Small: Used on set, folder, and class cards. Should be used as the default shadow for most elements.
  3. Medium: Used on menus, and navigation menu overlays like fly out menus and the sidebar.
  4. Large: Used on surfaces such as modal dialog backgrounds, set preview cards, and study mode backgrounds.

Behavior

  • With an interactive element like a card, we can transition between the card resting depth (small) and a hover depth (medium) to highlight the action.
  • When transitioning between depths, the element should look like it is being elevated higher when interacted with.
  • You should not have a white surface on a white background without a bordered container. To avoid confusion of identifying the two surfaces, add a var(--neutral-300) border to the element to create adequate separation.
