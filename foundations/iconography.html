Icons are visual representations of commands, files, devices, directories, or common actions, and are used to provide visual context to enhance usability. They should be simple, bold enough to grab attention, and memorable. We utilize Streamline Bold icons in our set, and create our own if necessary.

Icon sizes

Our icons should be displayed at two different sizes — 12px and 16px. We design our icons on a 16px canvas and exported as SVG to be resized with code. Our 16px size is the default usage 90% of the time so we optimize our icon library with this in mind. Some icons will not be perfectly crisp when using smaller and larger variations. When space is extreme tight and an icon is necessary, we utilize Streamline Micro Bold icons.

Creating new icons

Grid, layout, shapes, and keylines

Icons should be designed to be displayed at 16px, but are drawn on a 14px grid per the base Streamline set. Similarly, the 12px icons are displayed at 12px but drawn on a 10px grid. Designing icons this way allows for even spacing in the UI and uniformity within the icon set.

Create icons on the grid using basic foundational shapes, which include; circles, rectangles (horizontal and vertical), and squares. Sticking to these shapes in the grid keeps consistent proportions throughout the icon set.
