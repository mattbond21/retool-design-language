The Assembly Design Language encourages consistency across platforms and screen sizes by using uniform elements and spacing. This helps our customers move between devices without having to relearn the application each time.

By using our standard spacing, the user interface remains consistent, can be scaled easily, and uses rhythm to align elements correctly.

We utilize an 8px “soft grid” for base spacing and element sizes. We make an exception for using 4px as a half unit only. The below spacing increments apply to both horizontal and vertical spacing between elements.

It is important to use these spacing sizes as they are prescribed, and not to combine them. For example, 36px (combining 32px and 4px) should not be a measurement in our app. Furthermore, 56px should also not be used even though it is divisible by 8px.
